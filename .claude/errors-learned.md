## [2026-07-21] — Netlify DB no inyecta connection string en functions "Lambda compatibility"

**Contexto:** Fase 1, implementando `netlify/functions/brands.ts` (CRUD) usando `@netlify/database` (`getDatabase()`).
**Error:** `MissingDatabaseConnectionError: The environment has not been configured to use Netlify Database. You must supply the connectionString option when calling getDatabase()`. Ocurría en producción aunque el deploy mostraba "Migrations applied" y el panel Database confirmaba "Your database is ready!".
**Causa raíz:** La función estaba escrita en el formato clásico `export const handler: Handler = async (event) => {...}` (estilo Lambda/`@netlify/functions` v1). Netlify solo inyecta automáticamente la connection string de Netlify DB en el formato moderno de functions (`export default async (req: Request) => {...}`, basado en Web Request/Response). El formato clásico corre en "modo de compatibilidad Lambda" y no recibe esa inyección automática.
**Fix aplicado:** Se reescribieron `brands.ts` y `generate-batch.ts` al formato v2 (`export default async (req: Request) => new Response(...)`), usando `new URL(req.url).searchParams` en vez de `event.queryStringParameters` y `await req.json()` en vez de `JSON.parse(event.body)`. Tras el redeploy, `getDatabase()` funcionó sin pasar `connectionString` manualmente.
**Prevención:** En este proyecto, TODAS las Netlify Functions que usen `@netlify/database` deben escribirse en formato v2 (`export default`), nunca en formato clásico `Handler`. `health.ts` (que no toca la DB) se dejó en formato clásico porque funciona igual para ese caso simple, pero cualquier function nueva que use la DB debe partir del patrón v2 desde el inicio.
**Archivos:** `netlify/functions/brands.ts`, `netlify/functions/generate-batch.ts`
